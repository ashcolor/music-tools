# 振り子バーの動きについて

BPM円の下にある左右に動くバー（[Pendulum.tsx](../src/components/Pendulum.tsx)）の動きの計算式と実装メモ。

## 動きの仕様

- 1拍ごとに「中央 → 端 → 中央」の半周期を描く
- 拍の音が鳴る瞬間（`currentBeat` が切り替わる瞬間）に必ず中央
- 拍ごとに振れる方向を反転（奇数拍は右、偶数拍は左）
- 物理的な単振動（中央で最速、両端で速度0）を再現

## パラメータ

| 記号 | 意味 | 値 |
|---|---|---|
| `T` | 1拍の所要時間 | `60 / BPM` 秒 |
| `A` | 振幅（片側の最大変位） | `(TRACK_WIDTH - INDICATOR_WIDTH) / 2 = 5.625rem` |

トラック幅とインジケーター幅は [Pendulum.tsx](../src/components/Pendulum.tsx) の定数で定義。

## 位置の計算式

1拍内の進捗を `τ ∈ [0, 1]`（`τ = t / T`）とすると、位置は

```
x(τ) = A × sin(π × τ)
```

- `τ = 0`（拍の頭）: `x = 0`（中央）
- `τ = 0.5`（拍の中間）: `x = A`（端）
- `τ = 1`（次の拍）: `x = 0`（中央）

## 速度の計算式

### 平均速度（片道）

片道の所要時間が `T/2` なので

```
v_avg = A / (T/2) = 2A × BPM / 60 ≈ 0.1875 × BPM  [rem/s]
```

例: BPM=120 → 22.5 rem/s（≒ 360 px/s）

### 瞬間速度（単振動）

```
v(t) = dx/dt = A × (π / T) × cos(π × t / T)
```

- 最大速度（中央通過時、`τ = 0` または `τ = 1`）

  ```
  v_max = A × π / T = A × π × BPM / 60 ≈ 0.294 × BPM  [rem/s]
  ```

- 最小速度（端、`τ = 0.5`）: `0`

例: BPM=120 → `v_max ≈ 35.3 rem/s`

## CSS実装

`linear` タイミング関数 + キーフレームの折れ線近似で sin カーブを再現している（[main.css](../src/assets/main.css) の `@keyframes pendulum-right` / `pendulum-left`）。

- 5%刻み（20分割）でキーフレームを定義
- 各キーフレームの位置 = `var(--pendulum-swing) × sin(π × τ)`
- アニメーション全体は `linear` で再生 → キーフレーム間が線形補間され、sin カーブの折れ線近似になる

`var(--pendulum-swing)` はコンポーネントから振幅 `A` をCSS変数として渡している。

### キーフレームの値

| τ | sin(πτ) |
|---|---|
| 0.00 | 0.0000 |
| 0.05 | 0.1564 |
| 0.10 | 0.3090 |
| 0.15 | 0.4540 |
| 0.20 | 0.5878 |
| 0.25 | 0.7071 |
| 0.30 | 0.8090 |
| 0.35 | 0.8910 |
| 0.40 | 0.9511 |
| 0.45 | 0.9877 |
| 0.50 | 1.0000 |

`τ = 0.5` 以降は対称なので逆順で同じ値。

## 音タイミングとの同期

[MetronomeContext.tsx](../src/contexts/MetronomeContext.tsx) で `currentBeat` は `requestAnimationFrame` ループ内、`AudioContext.currentTime` がスケジュール済みクリック時刻に到達した瞬間に更新される。Pendulum コンポーネントはこの `currentBeat` を `key` に使ってアニメーションを再起動するため、音が鳴った瞬間にバーが中央スタートする形で同期する。

BPMが途中で変化（accel/decel モード）しても、各拍の頭で再起動されるため次の拍の頭にちょうど中央へ戻る。
